# Instructions for Codex

Работай итерациями. Не добавляй функциональность, которая не включена в текущую задачу.

Перед изменениями:
1. Изучи документы в `docs/`.
2. Кратко опиши план.
3. Не меняй доменную модель без явного согласования.
4. После реализации запусти тесты и обнови README.

## Защищенная документация

- Файлы `README.md`, `docs/vision.md`, `docs/domain-model.md` и `docs/architecture.md` нельзя изменять без прямого указания пользователя в текущей задаче.
- Не улучшай, не дополняй и не синхронизируй эти документы самостоятельно.
- При обнаружении противоречия сначала опиши проблему и запроси решение, не изменяя документы.
- Любое изменение доменной модели или архитектурного решения выполняй только в рамках отдельной явно сформулированной задачи.
