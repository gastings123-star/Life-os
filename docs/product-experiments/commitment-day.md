# Commitment Day + Evening Close

## Статус и гипотеза

Это временный vertical slice для семидневного dogfood-эксперимента, а не утверждённая
доменная модель Life OS. Гипотеза: короткий утренний договор об одном главном и не более чем
двух дополнительных результатах, дополненный обязательным вечерним пересмотром, ощущается и
работает иначе, чем обычный список задач.

Целевой сегмент эксперимента — один локальный пользователь, которому обычный todo-list помогает
сохранять задачи, но не помогает реалистично выбирать нагрузку и честно разбирать незавершённое.

## Архитектурное решение

Проект не переписывается с нуля: сохраняются слои Domain, Application, Infrastructure и API,
SQLite, текущие настройки и Docker-запуск. Существующие `Day`, `Action`, `Inbox` и их API остаются
без изменений, чтобы не потерять данные и не смешать продуктовый pivot с миграцией старого flow.

Эксперимент использует отдельный узкий aggregate:

- `DailyPlan` хранит дату, оценку capacity и состояние утреннего договора;
- `DailyCommitment` хранит выбранный результат и явно отделён от обычного `Action`;
- `CommitmentResolution` хранит неизменяемый итог закрытия, причину и связь с successor.

Это экспериментальные имена и границы. Универсальная сущность `Commitment`, связь с обычными
Actions, окончательная роль Day, модель времени/энергии и жизненный цикл после эксперимента
намеренно отложены. Полноценная модель Commitment понадобится только если dogfood подтвердит,
что ритуал регулярно используется, воспринимается не как todo-list и требует общей истории за
пределами одного дня.

## Пользовательский цикл

1. Утром пользователь выбирает локальную календарную дату и capacity: low, normal или high.
2. Формулирует один главный результат и, при необходимости, до двух дополнительных.
3. Подтверждает договор и в течение дня отмечает выполненные результаты.
4. Вечером для каждого результата явно выбирает completed, dropped или renegotiated.
5. Для dropped/renegotiated указывает причину; для other — комментарий.
6. При renegotiated явно выбирает дату. Только тогда создаётся новый successor с новым ID.
7. Закрытый день остаётся доступным только для чтения. Пропущенный прошлый close можно выполнить
   задним числом; автоматического переноса нет.

Дата всегда определяется клиентом в локальном часовом поясе и передаётся backend как `YYYY-MM-DD`.
Временные метки сохраняются в UTC. Запрос незакрытых дней принимает явную границу `before`, поэтому
backend не определяет пользовательское «сегодня» по часовому поясу сервера.

## Ограничения эксперимента

Нет авторизации, нескольких пользователей, уведомлений, проектов, целей, привычек, AI, недельного
планирования, аналитического dashboard, subtasks, приоритетов, тегов и автоматического переноса.
Capacity не является учётом часов или энергии. Старые todo-сценарии доступны по API, но скрыты с
главного экспериментального экрана.

## Метрики и данные

`GET /api/v1/experiments/commitment-day/summary` возвращает число запланированных и закрытых дней,
close rate, распределение итогов и причин за опциональный диапазон `date_from` / `date_to`.
Capacity, время закрытия, причины и successor links хранятся в БД и позволяют отдельно вычислить
число commitments на день и закрытия задним числом. Визуальный dashboard намеренно отсутствует.

Симулированные и тестовые данные не являются результатом реального discovery.

### Persevere

- close rate не ниже 60%;
- доля выполненных commitments выше baseline обычных задач;
- перенос происходит через явный renegotiation;
- пользователь возвращается к morning plan;
- ритуал воспринимается иначе, чем обычный todo-list.

### Pivot / Kill

- Evening Close массово пропускается;
- ограничение результатов только усиливает чувство вины;
- экран используется как обычный todo-list;
- ритуал не даёт ценности по сравнению с бумажным списком.

## Локальный dogfood

Запустить приложение:

```bash
docker compose up --build
```

Открыть <http://localhost:5173>, пройти утренний план, отметить выполненное и закрыть день. Для
проверки результатов:

```bash
curl "http://localhost:8000/api/v1/experiments/commitment-day/summary"
```

Остановить приложение:

```bash
docker compose down
```
