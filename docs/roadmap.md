# Life OS Roadmap

## Принципы

- Разработка идет небольшими итерациями.
- Каждая итерация должна завершаться рабочим состоянием проекта.
- Сначала инфраструктура, затем подтверждённый доменный срез, затем пользовательский сценарий.
- Одна итерация реализует только один согласованный сценарий.

---

## Phase 1. Foundation

Цель:
Подготовить технический фундамент проекта.

Завершённые итерации:

1. Создание технического каркаса проекта.
2. Добавление инфраструктуры хранения данных.

Результат:
Проект запускается локально и через Docker, имеет backend, frontend, тестовую инфраструктуру, SQLite, SQLAlchemy и Alembic без бизнес-моделей.

---

## Phase 2. Daily Planning Vertical Slice

Цель:
Реализовать первый подтверждённый пользовательский сценарий ежедневного планирования.

Следующая итерация:

1. Пользователь открывает выбранный календарный день.
2. Система получает существующий `Day` или создаёт новый.
3. Пользователь добавляет одно `Action`.
4. После повторного чтения действие отображается в том же дне.

Граница итерации:

- реализуются только `Day` и `Action` в объёме, зафиксированном в `docs/domain-model.md`;
- одна итерация должна реализовать только этот vertical slice;
- редактирование, удаление, выполнение, перенос, приоритеты, теги, цели, проекты, привычки, события, AI, авторизация и интеграции не входят в итерацию;
- последующие бизнес-функции не планируются детально до проверки первого среза.

Результат:
Минимальный сценарий работает сквозным образом через domain, application, persistence, HTTP API и frontend.

---

## Phase 3. Application

Цель:
Расширять пользовательские сценарии только после проверки первого vertical slice и отдельного согласования следующей итерации.

---

## Phase 4. Intelligence

Цель:
Добавление интеллектуальных возможностей системы после стабилизации базовых пользовательских сценариев.

---

## Phase 5. Ecosystem

Цель:
Интеграции, импорт/экспорт, резервное копирование и расширения после стабилизации ядра.

---

## Правило разработки

Каждая следующая задача должна реализовывать только одну небольшую итерацию roadmap.
Нельзя начинать следующую итерацию или расширять её границы без отдельного согласования.
