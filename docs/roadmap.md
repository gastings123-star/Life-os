# Life OS Roadmap

## Принципы

- Разработка идет небольшими итерациями.
- Каждая итерация должна завершаться рабочим состоянием проекта.
- Сначала инфраструктура, затем доменная модель, затем пользовательские сценарии.

---

## Phase 1. Foundation

Цель:
Подготовить технический фундамент проекта.

Итерации:

1. Создание технического каркаса проекта.
2. Настройка backend.
3. Настройка frontend.
4. Docker.
5. CI.
6. Тестовая инфраструктура.

Результат:
Проект запускается одной командой.

---

## Phase 2. Domain

Цель:
Реализовать базовую доменную модель.

---

## Phase 3. Application

Цель:
Реализовать первые пользовательские сценарии.

---

## Phase 4. Intelligence

Цель:
Добавление интеллектуальных возможностей системы.

---

## Phase 5. Ecosystem

Цель:
Интеграции, импорт/экспорт, резервное копирование, расширения.

---

## Правило разработки

Каждая следующая задача должна реализовывать только одну небольшую итерацию roadmap.
Нельзя начинать следующую фазу до завершения предыдущей.
